GraphicsMath

Introduction

GraphicsMath is a C++ header-only computer graphics math library. It has optional python bindings, and support for usage in CUDA device code.

Instead of using C++ templates, the source code of types, functions and their respective tests are generated from jinja2 template files. This is an effort to improve compilation times and code readability, at cost of extra source code and a discrete code-gen step.

Library usage

The library is organized into Types and Functions.

Building

A convenience build script is provided at the root of the repository for building all targets, and optionally installing to a location: ./build.sh <OPTIONAL_INSTALL_LOCATION>.

Building Python Bindings

To build python bindings for GraphicsMath, set the cmake option BUILD_PYTHON_BINDINGS="ON" when configuring cmake.

Developer Notes

Source Tree

The C++ headers are organized into the three categories:

  • src/gm/base - common utilities and definitions.
  • src/gm/types - computer graphics value types (vectors, arrays).
  • src/gm/functions - functionality which operate on gm types.

Python bindings for types and functions are available under src/gm/python.

Tests and benchmark code reside within the sub-directories.

Code Generation

The executable python script located at src/gm/genCode.py can be run to generate and update source code. Coe-formatters clang-format and black need to be available in the codegen environment.

Some common code generation logic are located under src/gm/codeGen/.

The jinja2 templates are located under src/gm/template.

Motivation

Why another math library?

This is a personal project to internalize math concepts and explore data and compute optimization techniques.
